config/database.sql

config/database.sql

MySQL database schema that creates all required tables.

Tables Created:
  • users - User accounts (id, username, email, password_hash, display_name, avatar, is_online)
  • sessions - User sessions (token, expires_at)
  • peer_connections - Peer connections (user_one_id, user_two_id, status)
  • messages - Chat messages (sender_id, receiver_id, content, message_type)
  • ice_candidates - WebRTC ICE candidates for signaling
  • online_peers - Currently online users (user_id, socket_id, peer_id)

public/autoload.php

public/autoload.php

Simple PSR-4 style autoloader that loads PHP classes on demand.

spl_autoload_register(function ($class) {
    $prefix = 'P2P\\';
    $baseDir = __DIR__ . '/../src/';
    
    $len = strlen($prefix);
    if (strncmp($prefix, $class, $len) !== 0) {
        return;
    }
    
    $relativeClass = substr($class, $len);
    $file = $baseDir . str_replace('\\', '/', $relativeClass) . '.php';
    
    if (file_exists($file)) {
        require $file;
    }
});

src/Core/Config.php

src/Core/Config.php

Configuration manager class that loads and provides access to configuration values.

Key Methods:
Method Description
Config::init() Initialize configuration from file
Config::get($key, $default) Get config value (supports dot notation)
Config::set($key, $value) Set config value
Usage:
// Get database host
$host = Config::get('database.host', 'localhost');

// Get nested value
$jwtSecret = Config::get('auth.jwt_secret');

Database::fetch($sql, $params) Fetch single row

src/Core/Router.php

src/Core/Router.php

HTTP request router that maps URLs to handler functions.

Usage:
$router = new Router();

// Define routes
$router->get('/api/users', [$this, 'getUsers']);
$router->post('/api/users', [$this, 'createUser']);
$router->put('/api/users/{id}', [$this, 'updateUser']);
$router->delete('/api/users/{id}', [$this, 'deleteUser']);

// Dispatch request
$router->dispatch($_SERVER['REQUEST_METHOD'], $_SERVER['REQUEST_URI']);

src/Auth/Authenticator.php

Handles user authentication (registration, login, logout).

Key Methods:

  • login($username, $password) - Authenticate user

  • validateToken($token) - Verify JWT token

server/websocket.php

server/websocket.php

WebSocket server for real-time messaging and WebRTC signaling.

Key Features:
  • User registration and connection management
  • Real-time message forwarding
  • WebRTC signaling (offer, answer, ICE candidates)
  • Typing indicators
  • User online/offline status broadcasting
Message Types:
{
  "type": "register",      // Register user with socket
  "type": "message",       // Chat message
  "type": "typing",        // Typing indicator
  "type": "offer",         // WebRTC offer
  "type": "answer",        // WebRTC answer
  "type": "ice-candidate"  // WebRTC ICE candidate
}
