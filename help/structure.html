📋 Folder Purpose Details

📂 config/

Contains all configuration files for the application.

  • config.php - Main application configuration (database, auth, WebSocket, P2P settings)
  • database.sql - MySQL database schema with all tables and sample data

📂 public/

The web root directory that is served to users. Contains all frontend files.

  • index.php - Main entry point that handles all HTTP requests
  • index.html - Single-page application HTML
  • autoload.php - PSR-4 style class autoloader
  • css/styles.css - All application styling
  • js/app.js - Frontend JavaScript with WebSocket and WebRTC

📂 server/

Contains server-side programs that run separately from the web server.

  • websocket.php - WebSocket server for real-time messaging and signaling

📂 src/

PHP source code following PSR-4 autoloading standards.

src/Auth/
  • JWT.php - JSON Web Token generation and validation
  • Authenticator.php - User registration, login, logout handling
src/Controllers/
  • ApiController.php - All REST API endpoints
src/Core/
  • Config.php - Configuration loader and manager
  • Database.php - PDO database connection wrapper
  • Router.php - HTTP request router
src/Models/
  • User.php - User data operations
  • Message.php - Message storage and retrieval
  • Peer.php - Peer connections management

📂 logs/

Application log files for debugging and monitoring.

📂 uploads/

Directory for user-uploaded files (if implementing file upload feature).

🛠️ Technology Stack

Layer Technology Description
Frontend HTML5, CSS3, JavaScript User interface
Backend PHP 8.2+ Server logic
Database MySQL 5.7+ Data storage
Real-time WebSocket (PHP Sockets) Instant messaging
P2P WebRTC Direct file transfer
Auth JWT (JSON Web Tokens) Secure authentication

🏗️ MVC Architecture

This application follows the Model-View-Controller (MVC) design pattern:

Component Location Responsibility
Model src/Models/ Data operations and business logic
View public/ User interface (HTML, CSS, JS)
Controller src/Controllers/ Request handling and response
